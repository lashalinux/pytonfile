Git გლობალური და ლოკალური კონფიგურაცია
Git-ს აქვს ორი ძირითადი კონფიგურაცია:

1. გლობალური კონფიგურაცია
ეს მოქმედებს ყველა პროექტზე, რაც შენს კომპიუტერზეა. მაგ:

bash
კოდის კოპირება
git config --global user.name "შენი სახელი"
git config --global user.email "example@email.com"
ეს ინფორმაცია ავტომატურად დაემატება ყველა ახალ Git პროექტს.

2. ლოკალური კონფიგურაცია
ეს მოქმედებს მხოლოდ კონკრეტულ პროექტზე, იმ ფოლდერში სადაც .git არის.

bash
კოდის კოპირება
git config user.name "პროექტის მომხმარებელი"
git config user.email "პროექტის@იმეილი.com"
ამით შეგიძლია ერთზე მეტი იდენტობით იმუშაო (მაგ. პირადი და სამუშაო).

Git პროექტის ინიციალიზაცია
შედი პროექტის ფოლდერში:

bash
კოდის კოპირება
cd ჩემი_პროექტი/
გაუშვი Git ინიციალიზაცია:

bash
კოდის კოპირება
git init
ეს შექმნის .git ფოლდერს — Git იწყებს ცვლილებების ტრეკინგს.

დაამატე ფაილები და შეინახე commit-ით:

bash
კოდის კოპირება
git add .
git commit -m "პირველი კომიტი"
დამატებით: კონფიგურაციის ნახვა
გლობალური კონფიგურაცია:

bash
კოდის კოპირება
git config --global --list
ლოკალური (პროექტის) კონფიგურაცია:

bash
კოდის კოპირება
git config --list
