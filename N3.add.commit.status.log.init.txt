Git-ის საბაზო ცნებები და ბრძანებები
1. git init — პროექტის ინიციალიზაცია
Git-ს ეუბნები: „აი, ეს პროექტი მინდა ვმართო შენით“.

bash
კოდის კოპირება
git init
2. git status — სტატუსის შემოწმება
გიჩვენებს:

რომელი ფაილები შეიცვალა

რომელი ფაილებია stage-ზე (დამატებულია commit-ისთვის)

რომელი ჯერ არ არის დამატებული

bash
კოდის კოპირება
git status
3. git add — ფაილის stage-ზე დამატება
Git-ს ეუბნები: „ეს ფაილი მზადაა commit-ისთვის“.

ერთ ფაილზე:

bash
კოდის კოპირება
git add file.txt
ყველა ფაილზე:

bash
კოდის კოპირება
git add .
4. git commit — ცვლილებების შენახვა
გაკეთებული ცვლილებების შენახვა Git-ის ისტორიაში.

bash
კოდის კოპირება
git commit -m "შეტყობინება ცვლილების შესახებ"
მაგ:

bash
კოდის კოპირება
git commit -m "პირველი ვერსია: index.html დამატებულია"
5. git log — ცვლილებების ისტორიის ნახვა
გიჩვენებს ყველა commit-ს: ვინ გააკეთა, როდის და რას შეეხებოდა.

bash
კოდის კოპირება
git log
უფრო მოკლედ (ერთი ხაზი თითო commit-ზე):

bash
კოდის კოპირება
git log --oneline
მოკლე პროცესი პრაქტიკაში:
bash
კოდის კოპირება
git init                    # Git ინიციალიზაცია
git status                 # ვნახოთ სტატუსი
git add file.txt           # დავამატოთ ფაილი stage-ზე
git commit -m "დამატებულია file.txt"   # შევინახოთ ცვლილება
git log                    # ვნახოთ
